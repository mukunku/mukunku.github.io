Intro

Today we will be developing a simple application workflow to showcase how structured performance data is gathered, loaded, analyzed and finally delivered/utilized. While this workflow is not all we do as the Structured Data Team, it is our bread and butter. {

	System.Net.WebRequest request = System.Net.WebRequest.Create(websiteUrl);

	string html = string.Empty;
	using (System.Net.WebResponse response = request.GetResponse())
	{
		Stream responseStream = response.GetResponseStream();
		using (StreamReader reader = new StreamReader(responseStream))

		}

        private static DataTable ParseReport(string report)
        {
            DataTable result = GenerateEmptySchema();

            using (TextFieldParser parser = new TextFieldParser(report))
            {
                parser.TextFieldType = FieldType.Delimited;
                parser.SetDelimiters(",", "|", "\t"); //Common delimiters for these files are commas, pipes and tabs
                parser.TrimWhiteSpace = true;
                while (!parser.EndOfData)
                {
                    DataRow row = result.NewRow();

                    string[] fieldRow = parser.ReadFields();
                    int fieldIndex = 0;
                    foreach (string fieldRowCell in fieldRow)
                    {
                        if (!string.IsNullOrWhiteSpace(fieldRowCell))
                        {
                            if (result.Columns[fieldIndex].DataType == typeof(DateTime))
                            {
                                DateTime value = DateTime.MinValue;
                                Match dateMatch = Regex.Match(fieldRowCell, "(1|2)\\d{7}");
                                if (dateMatch.Success
                                    && DateTime.TryParseExact(dateMatch.Value, "yyyyMMdd", new CultureInfo("en-US"), DateTimeStyles.AssumeLocal, out value))
                                {
                                    row[fieldIndex] = value;
                                }
                                else
                                    throw new Exception("Invalid DateTime value");
                            }
                            else if (result.Columns[fieldIndex].DataType == typeof(float))

                            }

                }

        private static void LoadToDB(DataTable table, string targetTable)
        {
            using (SqlBulkCopy bulkCopy = new SqlBulkCopy(DB_CONNECTION_STRING))
            {
                //Warning column names are case sensitive
                bulkCopy.DestinationTableName = targetTable;
                bulkCopy.BatchSize = 1000; //Load 1000 records at a time

                bulkCopy.WriteToServer(table);

Optimization

We have our view that is performing some aggregation. Let's analyze the query's performance by enabling the Query Plan Analysis and re-running our SELECT statement

We can see that the query is scanning the entire table to obtain the required values for our aggregation. Right now that isn't a big deal since this table is only housing data for a single deal. But in our production database, where we have more than 30 million records, this would be unacceptable performance as it would take several minutes to complete. In order to optimize the operation we need to create an Index. But on which columns?
